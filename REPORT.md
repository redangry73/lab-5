# Лабораторная работа №5. Git Advanced

## Цель работы

Освоить работу с ветками, слиянием, моделированием и разрешением merge-конфликтов в Git.

## Ход работы

### 1. Клонирование репозитория

```bash
git clone <URL_вашего_репозитория>
cd <имя_репозитория>
```

### 2. Добавление файла и первый коммит в `main`

```bash
cat > README.md << 'EOF'
# Название книги

## Глава 1: Введение
Здесь будет введение в тему книги.

## Глава 2: Основы Git
Основные понятия и команды Git.
EOF

git add README.md
git commit -m "Initial README with chapters 1 and 2"
git push origin main
```

### 3. Работа в новой ветке `feature-login`

```bash
git checkout -b feature-login
cat > README.md << 'EOF'
# Название книги

## Глава 1: Введение
Здесь будет введение в тему книги.

## Глава 2: Основы Git
Основные понятия и команды Git.

## Глава 3: Вход в систему
Раздел по новой функциональности входа в систему.
EOF

git add README.md
git commit -m "Добавлена глава 3: Вход в систему"
git push origin feature-login
```

### 4. Изменения в `main`

```bash
git checkout main
cat > README.md << 'EOF'
# Название книги: Приключения в мире Git

## Глава 1: Введение
Здесь будет введение в удивительный мир Git.

## Глава 2: Основы Git
Основные понятия и команды Git.
EOF

git add README.md
git commit -m "Изменено название книги и введение"
git push origin main
```

### 5. Моделирование конфликта в `feature-login`

```bash
git checkout feature-login
cat > README.md << 'EOF'
# Название книги: Приключения в мире Git

## Глава 1: Введение
Здесь будет введение в удивительный мир Git.

## Глава 2: Основы Git и магия конфликтов
Основные понятия и команды Git, а также волшебство разрешения конфликтов.

## Глава 3: Вход в систему
Раздел по новой функциональности входа в систему.
EOF

git add README.md
git commit -m "Добавлен раздел о магии конфликтов"
git push origin feature-login
```

### 6. Слияние и разрешение конфликта

```bash
git checkout main
git pull origin main
git merge feature-login
```

После конфликта вручную оставлен итоговый вариант файла:

```md
# Название книги: Приключения в мире Git

## Глава 1: Введение
Здесь будет введение в удивительный мир Git.

## Глава 2: Основы Git и магия конфликтов
Основные понятия и команды Git, а также волшебство разрешения конфликтов.

## Глава 3: Вход в систему
Раздел по новой функциональности входа в систему.
```

Фиксация разрешения конфликта:

```bash
git add README.md
git commit -m "Resolved conflict in chapter 2"
git push origin main
```

## Результат

1. Создана отдельная ветка для новой функциональности.
2. Выполнено слияние веток.
3. Смоделирован конфликт изменений в одном участке файла.
4. Конфликт корректно разрешен и изменения отправлены в `main`.

## Что приложено в отчет

1. Скрин `git branch` (ветки `main`, `feature-login`).
2. Скрин конфликта в `README.md` (метки `<<<<<<<`, `=======`, `>>>>>>>`).
3. Скрин финального `README.md` после разрешения.
4. Скрин `git log --oneline --graph --all`.
